# Smart Liquidity 方法说明

## 核心流程

1. 并发连接 Binance、Bybit、OKX 的 USDT 本位永续合约接口
2. 拉取订单簿、Funding Rate、Open Interest
3. 对订单簿执行 5% 窗口反幌骗过滤
4. 将买卖盘按价格区间聚合
5. 以 USDT 名义价值排序，输出 Top 3 关键档位
6. 汇总 OI 与平均资金费率，推导多空情绪

## 关键口径

- `卖单总量 / 买单总量`：基础资产单位，例如 BTC
- `卖单名义价值 / 买单名义价值`：`价格 * 数量` 后得到的 USDT 价值
- `OI`：按基础资产单位跨交易所求和
- `OI USDT 估值`：`OI * 标记价格`

## 反幌骗规则

- 默认仅保留参考价格上下 5% 范围内的报价
- `--threshold` 存在时，用它替代盘口中间价作为过滤基准

## 降级规则

- 任一交易所订单簿失败，不影响其他交易所继续输出
- 任一交易所 Funding / OI 失败，不影响已有行情汇总
- 最终输出必须显示成功来源与失败来源
