# crypto-skills-cli

加密货币市场分析命令行工具集，面向 HFT 级精度的量化交易决策支持。

## 项目定位

本项目是一个可扩展的加密货币分析框架，以 CLI 和 MCP 形式持续集成各类市场分析能力。所有分析结果面向真实交易决策，**准确性**和**抗欺骗性**为最高优先级。

当前已实现的分析能力聚焦于**永续合约流动性研判**，后续将持续扩展更多 Skill 和 MCP 模块。

## 安装

```bash
# 确保 Python >= 3.10
pip install -r requirements.txt
```

程序会优先读取 `CRYPTO_PROXY`，其次读取常见系统代理环境变量；若都未配置，会尝试自动探测本地 `127.0.0.1:7897` / `127.0.0.1:7890` 代理端口。

如需手动指定代理，可显式设置环境变量 `CRYPTO_PROXY`，例如：

```bash
set CRYPTO_PROXY=http://127.0.0.1:7897
```

如需强制直连，可设置：

```bash
set CRYPTO_PROXY=none
```

## 快速开始

```bash
# 查看可用命令
python main.py --help

# 分析 BTC 流动性（参考价格 84000，区间 100 USDT）
python main.py smart-liquidity --symbol BTC --threshold 84000 --bin-size 100

# 分析 ETH 流动性（使用默认参数）
python main.py smart-liquidity --symbol ETH
```

### 分层说明

- **`core/`** — 可复用基础设施层。`FuturesAggregator` 使用 `ccxt.async_support` 异步并发连接三家交易所，提供统一的数据接入接口。新 Skill 应复用此层。
- **`skills/`** — 业务分析层。每个 Skill 封装完整的分析流程，独立可测。
- **`main.py`** — CLI 交互层。使用 typer 注册子命令，rich 渲染终端输出面板。
- **降级策略** — 任一交易所订单簿或行情接口失败时，CLI 会明确标注可用来源与失败原因，不再误报“已聚合三家交易所”。

### 数据来源

通过 `ccxt` 异步并发接入以下三家交易所的 USDT 本位永续合约 API：

| 交易所 | 合约类型 | ccxt 配置 |
|--------|----------|-----------|
| Binance | USDT-M 永续合约 | `defaultType: 'future'` |
| Bybit | 线性永续合约 | `defaultType: 'linear'` |
| OKX | 永续掉期合约 | `defaultType: 'swap'` |

`format_symbol()` 函数自动将用户输入（如 `btc`、`SOL`）转换为各交易所对应的合约符号格式。

## 已实现 Skill

### Smart Liquidity — 全网聚合流动性分析

**目的：** 通过聚合多交易所的真实订单簿深度，识别关键的压力位和支撑位，辅助交易者判断短期价格方向和入场/出场时机。

**详细文档：** [docs/skills/smart_liquidity.md](docs/skills/smart_liquidity.md)

#### 使用的核心指标

| 指标 | 获取方式 | 含义与意义 |
|------|----------|------------|
| **L2 订单簿深度** | 通过 `fetch_order_book()` 从三家交易所并发拉取 | 反映市场上真实的买卖挂单分布。通过聚合三家交易所数据，消除单一交易所的局限性，获得更准确的全网流动性图景。 |
| **反幌骗过滤后的挂单** | 对原始订单簿应用 5% 偏离阈值过滤 | 仅保留参考价格上下 5% 范围内的报价，排除远离盘口的可疑挂单，降低幌骗（Spoofing）带来的误导。 |
| **价格档位聚合** | 按用户指定的区间大小（默认 100 USDT）对订单簿分档 | 将零散的挂单聚合为价格区间，直观展示买卖压力集中的关键价位。Top 3 区间即为最关键的压力位/支撑位。 |
| **未平仓量 (OI)** | 通过 `fetch_open_interest()` 从三家交易所拉取后求和 | 反映当前市场中未结算的合约总量。OI 越高意味着杠杆资金越密集，价格波动可能引发更大规模的清算连锁反应。以基础资产单位（如 BTC）累加，展示时乘以标记价格换算为 USDT 价值。 |
| **资金费率 (Funding Rate)** | 通过 `fetch_funding_rate()` 从三家交易所拉取后取平均 | 永续合约特有的多空平衡机制。正费率表示多头向空头支付费用（多头拥挤），负费率则相反。极端费率可能预示价格反转。 |
| **多空情绪** | 基于平均资金费率计算 | > 0.01% 判定为看多（多头占优）、< -0.01% 为看空（空头占优）、否则中性。提供市场情绪的快速判断参考。 |

#### 命令参数

```bash
python main.py smart-liquidity --symbol <币种> [--threshold <参考价格>] [--bin-size <区间大小>]
```

| 参数 | 缩写 | 说明 | 默认值 |
|------|------|------|--------|
| `--symbol` | `-s` | 交易标的（如 BTC、ETH、SOL） | 必填 |
| `--threshold` | `-t` | 参考价格，覆盖盘口中间价作为反幌骗过滤基准 | 可选 |
| `--bin-size` | `-b` | 价格聚合区间大小（USDT） | 100 |

#### 输出面板

1. **全网聚合卖盘挂单量 Top 3 价格区间** — 显示每个价位区间的基础资产挂单量、USDT 名义价值和来源交易所
2. **全网聚合买盘挂单量 Top 3 价格区间** — 显示真实买盘支撑的数量与名义价值
3. **合约持仓量 (OI) 与多空情绪** — 标记价格、全网 OI（基础资产 + USDT 估值）、平均资金费率、多空情绪判定及解读
4. **交易所可用性提示** — 明确区分盘口来源、行情/OI 来源，以及失败接口的具体错误

#### 使用场景

- **短线交易**：识别关键压力位和支撑位，制定入场/出场策略
- **仓位管理**：通过 OI 判断市场杠杆密度，评估清算风险
- **情绪监控**：通过资金费率追踪多空力量变化，预判趋势反转
- **防操纵**：反幌骗过滤帮助识别真实挂单，避免被虚假深度误导
